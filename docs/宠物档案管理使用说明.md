# 宠物档案管理和健康日志模块使用说明

## 📋 功能概述

本模块实现了完整的宠物档案管理和健康日志跟踪系统,主要包括以下功能:

### 1. 宠物档案管理 🐾
- **录入**: 创建宠物档案,记录基本信息、识别信息、领养信息等
- **编辑**: 更新宠物信息
- **查询**: 查看宠物列表和详细信息
- **归档**: 归档不再活跃的宠物档案
- **删除**: 永久删除宠物档案

### 2. 健康记录 💉
- **疫苗接种记录**: 记录疫苗类型、接种日期、下次接种时间
- **驱虫记录**: 体内外驱虫记录
- **体检记录**: 定期健康检查记录
- **疾病治疗**: 疾病诊断、治疗方案、用药记录
- **手术记录**: 手术信息和恢复情况
- **其他医疗记录**: 自定义医疗记录

### 3. 健康提醒 🔔
- **定时提醒**: 设置疫苗、驱虫、体检等定时提醒
- **重复提醒**: 支持每天、每周、每月、每年重复
- **提前提醒**: 可设置提前几天开始提醒
- **完成标记**: 完成后可标记提醒完成

### 4. 体重跟踪 ⚖️
- 记录宠物体重变化
- 查看体重趋势
- 添加备注说明

### 5. 日常日志 📔
- 记录每日饮食情况
- 记录饮水、排泄情况
- 记录活动量和情绪
- 记录异常症状

## 🚀 快速开始

### 1. 数据库设置

在 Supabase 中执行 SQL 文件创建所需的数据表:

```bash
# 在 Supabase SQL Editor 中执行
database/pet-profile-tables.sql
```

这将创建以下表:
- `user_pets` - 用户宠物档案表
- `health_records` - 健康记录表
- `health_reminders` - 健康提醒表
- `weight_records` - 体重记录表
- `daily_logs` - 日常日志表

### 2. 访问页面

启动开发服务器后,访问以下路径:

- 宠物列表: `/my-pets`
- 添加宠物: `/my-pets/add`
- 宠物详情: `/my-pets/{pet-id}`
- 编辑宠物: `/my-pets/{pet-id}/edit`
- 添加健康记录: `/my-pets/{pet-id}/health/add`
- 添加健康提醒: `/my-pets/{pet-id}/reminder/add`

### 3. 导航集成

在主导航中添加"我的宠物"入口:

```tsx
// 在导航组件中添加
<Link href="/my-pets" className="nav-item">
  🐾 我的宠物
</Link>
```

## 💡 使用示例

### 添加宠物档案

1. 访问 `/my-pets`
2. 点击"添加宠物"按钮
3. 填写宠物信息:
   - 必填: 宠物名字
   - 选填: 品种、性别、出生日期、体重等
4. 点击"保存"

### 添加健康记录

1. 在宠物详情页点击"健康记录"标签
2. 点击"添加记录"
3. 选择记录类型(疫苗/驱虫/体检等)
4. 填写相关信息
5. 如果是疫苗或驱虫,可以设置下次时间
6. 点击"保存"

### 设置健康提醒

1. 在宠物详情页点击"健康提醒"标签
2. 点击"添加提醒"
3. 选择提醒类型
4. 设置提醒日期
5. 可选设置重复周期和提前提醒天数
6. 点击"保存"

### 记录体重

1. 在宠物详情页点击"体重记录"标签
2. 点击"添加记录"
3. 输入体重和日期
4. 可添加备注
5. 点击"保存"

### 添加日常日志

1. 在宠物详情页点击"日常日志"标签
2. 点击"添加日志"
3. 选择日期
4. 记录饮食、饮水、排泄、活动、情绪等
5. 可添加异常症状和备注
6. 点击"保存"

## 🎨 页面功能说明

### 宠物列表页面

- 展示所有宠物卡片
- 显示宠物头像、名字、品种、年龄、体重
- 提供"详情"、"编辑"、"归档"、"删除"操作
- 支持筛选显示已归档宠物

### 宠物详情页面

包含5个标签页:

1. **基本信息**: 显示宠物的所有档案信息
2. **健康记录**: 按时间倒序显示所有健康记录
3. **健康提醒**: 显示待完成和已完成的提醒
4. **体重记录**: 显示体重变化记录
5. **日常日志**: 显示每日健康日志

## 📊 数据库设计

### 核心表结构

#### user_pets (宠物档案表)
- 基本信息: 名字、品种、性别、出生日期等
- 识别信息: 芯片号、注册号
- 状态: active(活跃) / archived(归档) / deceased(已故)

#### health_records (健康记录表)
- 记录类型: vaccine/deworming/checkup/illness/surgery/other
- 医疗信息: 兽医、诊所、诊断、治疗、用药
- 附件: 支持上传报告、化验单等

#### health_reminders (健康提醒表)
- 提醒类型: vaccine/deworming/checkup/medication/grooming/custom
- 时间设置: 提醒日期、重复类型、重复间隔
- 状态: 启用/禁用、完成/未完成

#### weight_records (体重记录表)
- 体重值和单位(kg/lb)
- 记录日期
- 备注

#### daily_logs (日常日志表)
- 饮食和饮水情况
- 排泄情况
- 活动量和情绪
- 异常症状

## 🔐 安全性

- 使用 Row Level Security (RLS) 确保用户只能访问自己的数据
- 所有表都启用了 RLS 策略
- 通过 user_id 和 auth.uid() 进行权限控制

## 🛠️ API 函数说明

### 宠物档案 API (`userPets.ts`)

```typescript
// 创建宠物
createUserPet(petData)

// 获取用户的宠物列表
getUserPets(userId, includeArchived?)

// 获取单个宠物详情
getUserPetById(petId)

// 更新宠物信息
updateUserPet(petId, updates)

// 归档宠物
archiveUserPet(petId)

// 删除宠物
deleteUserPet(petId)

// 搜索宠物
searchUserPets(userId, query)

// 恢复归档的宠物
restoreUserPet(petId)
```

### 健康记录 API (`healthRecords.ts`)

```typescript
// 健康记录
createHealthRecord(recordData)
getHealthRecords(petId, recordType?)
updateHealthRecord(recordId, updates)
deleteHealthRecord(recordId)

// 健康提醒
createHealthReminder(reminderData)
getHealthReminders(petId, includeCompleted?)
getUpcomingReminders(petId, days?)
updateHealthReminder(reminderId, updates)
completeReminder(reminderId)
deleteHealthReminder(reminderId)

// 体重记录
createWeightRecord(recordData)
getWeightRecords(petId)
deleteWeightRecord(recordId)

// 日常日志
createDailyLog(logData)
getDailyLogs(petId, startDate?, endDate?)
getDailyLogByDate(petId, date)
updateDailyLog(logId, updates)
deleteDailyLog(logId)
```

## ❓ 常见问题

### Q: 如何恢复已归档的宠物?
A: 在宠物列表页勾选"显示已归档宠物",找到对应宠物后使用恢复功能。

### Q: 提醒会自动触发吗?
A: 目前需要用户手动查看,未来可以集成邮件或推送通知。

### Q: 可以为同一只宠物设置多个提醒吗?
A: 可以,每个提醒都是独立的,可以设置多个不同类型的提醒。

### Q: 数据是否安全?
A: 使用 Supabase 的 RLS 策略,每个用户只能访问自己的宠物数据,确保数据安全。
